## Decisions Made

1. **Frame-based playback at 60 FPS** — Integrates with `pygame.time.Clock()` loop, no new threads, testable
2. **Millisecond timestamps** — Precise enough for 60 FPS (±2ms tolerance) with simple state machine
3. **JSON timeline format** — Human-readable, versioned schema (v1.0), supports nested playback
4. **Flat file storage** — All recordings in `~/.mr-pumpkin/recordings/` with unique filenames
5. **Callback pattern for commands** — Decouples timeline module from PumpkinFace internals, enables testing
6. **Fail-fast on invalid commands** — Stop playback immediately on error, prevents cascading failures
