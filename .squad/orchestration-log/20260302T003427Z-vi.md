## Implementation Details

### 5-Phase Workflow (Both Scripts)

1. **Check:** Compare local VERSION vs. GitHub API latest release
2. **Download:** Fetch ZIP via gh CLI or direct URL
3. **Stop:** Gracefully stop pumpkin_face.py if running
4. **Deploy:** Extract, validate, copy files, run pip install
5. **Restart:** Launch with original command line arguments

### Key Patterns

**Version Comparison:**
- Parse semantic versioning (major.minor.patch)
- Compare as integer tuples for correct ordering (0.5.9 < 0.5.10)
- Strip 'v' prefix and whitespace

**Download Strategy:**
- Check for `gh` command availability
- Use gh CLI when available (better authentication)
- Fallback to curl/Invoke-WebRequest for direct download

**Process Detection:**
- Bash: `pgrep -f "[p]ython.*pumpkin_face.py"` (bracket trick avoids grep match)
- PowerShell: `Get-WmiObject Win32_Process | Where-Object { $_.CommandLine -like "*pumpkin_face.py*" }`

**Graceful Stop:**
- Bash: SIGTERM, wait 5 seconds, SIGKILL if needed
- PowerShell: Stop-Process, wait 5 seconds, Stop-Process -Force

**Validation Before Deploy:**
- Check for required files (pumpkin_face.py, VERSION, requirements.txt)
- Verify ZIP integrity before extraction
- Abort if any required file missing

**Logging:**
- Append-only log file: `mr-pumpkin-update.log`
- Format: `YYYY-MM-DD HH:MM:SS | PHASE | Message`
- Includes all operations (check, download, stop, deploy, restart)

## Design Decisions

**External Scripts vs. Embedded Logic**
- External approach enables cron/Task Scheduler scheduling
- Separation of concerns: graphics app doesn't manage updates
- Can modify main app while stopped

**Platform-Specific Scripts (bash + PowerShell)**
- Windows file locking vs. Linux inode swap require different approaches
- Native tools more reliable than cross-platform abstraction
- Follows existing install.sh/install.ps1 convention

**Idempotent Design**
- Safe to run from cron jobs
- Multiple runs produce same result
- "Already up-to-date" message when no update available

## Insights & Learnings

**Release Infrastructure Patterns:** Existing install scripts provide proven patterns for dependency checking, error handling, and cross-platform support. Update scripts follow same conventions for consistency.

**Process Handling Complexity:** Windows process detection requires WMI queries (Get-Process doesn't include CommandLine by default in PowerShell 5.1). Linux pgrep with bracket trick avoids matching grep itself.

**Download Strategy Tradeoffs:** gh CLI provides better authentication and rate limit handling, but requires installation. Direct URL fallback ensures script works on systems without gh.

**Cron Job Environment:** Scripts must set up PATH and export necessary variables (cron runs with minimal environment). Absolute paths and explicit PATH exports required.
