## Key Testing Patterns

**Semantic Version Comparison:**
- Parse as tuple of integers: `(major, minor, patch)`
- Direct tuple comparison (Python handles element-wise)
- Strip 'v' prefix and whitespace before parsing

**ZIP Validation:**
- Use `zipfile.ZipFile.namelist()` to read contents
- Check for basename matches (handles subdirectories)
- BadZipFile exception catches corrupted files

**File Deployment with Preservation:**
- Pattern matching: `item.startswith('timeline_') and item.endswith('.json')`
- Skip user data files during deployment
- Return list of copied files for verification

**JSON Parsing:**
- Handle missing fields gracefully (raise ValueError)
- Detect malformed JSON (raise JSONDecodeError)
- Strip whitespace from extracted values

## Insights & Learnings

**Testing Shell Logic via Python:** Shell scripts are difficult to unit test. Python helper functions validate core logic and serve as reference implementation for shell translation.

**Semantic Version as Standard:** Treating versions as tuples of integers enables correct comparison (0.5.9 < 0.5.10 as numeric, not string). This is critical for update logic.

**Preservation Pattern:** File deployment logic must distinguish user data (timeline_*.json) from application files (pumpkin_face.py, requirements.txt). This prevents loss of user recordings.

**JSON Parsing Robustness:** GitHub API responses may have variations. Explicit error handling for missing fields and malformed JSON prevents silent failures.
