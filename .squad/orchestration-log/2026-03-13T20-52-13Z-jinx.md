## Outcome

**Decided Raspberry Pi should use hybrid apt/pip install behavior and keep update.sh non-root.**

### Key Decisions

1. **Hybrid dependency strategy on Raspberry Pi:**
   - install.sh should detect Raspberry Pi and prefer apt for known problematic Python/runtime dependencies
   - install.sh should still use pip for remaining PyPI-only dependencies
   - update.sh should remain non-root and should not require apt upgrade behavior

2. **Rationale:**
   - Reliability: Some Python packages are more dependable from distro repos on Pi hardware
   - Operational safety: Auto-update designed for unattended cron use; requiring sudo apt-get breaks that model
   - Versioning reality: Not all dependencies are realistically available via apt at suitable versions

3. **Guidance for implementation:** Split dependency install/update paths into apt-managed and pip-managed groups on Raspberry Pi, keeping update.sh non-root by default.
