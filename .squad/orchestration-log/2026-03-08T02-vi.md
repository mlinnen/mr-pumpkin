## Key Technical Decisions

| Decision | Rationale |
|----------|-----------|
| Base64 encoding (not file upload) | Simpler than Gemini's upload-poll-delete lifecycle; single API call |
| `gpt-4o` (not gpt-4-turbo) | Strong JSON instruction-following; multimodal audio support |
| `gpt-4o-audio-preview` model | Supports inline audio analysis; simpler than separate transcription API |
| Environment variable pattern | Matches existing Gemini provider pattern (`GEMINI_API_KEY` → `OPENAI_API_KEY`) |

## Rationale

**Why now:** Gemini quota exhaustion is blocking users. OpenAI provides quota resilience without code changes to the API contract (both implement existing ABCs).

**Why OpenAI:** 
- Simpler audio handling than Gemini (base64 inline vs file upload)
- Strong JSON output quality from `gpt-4o`
- Proven multimodal audio support
- Factory pattern allows future providers (Claude, etc.)

**Why both passes:** Audio analysis is where quota exhaustion happens most (larger model, file overhead). Providing both providers at once eliminates worst-case blocking.
